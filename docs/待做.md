## A1. InterGNN 加深（欠传播修复） ✅ v1.0.0 已实施

### 动机
`InterGNN` 当前只有 2 层 SAGEConv，感受野 = 2 跳。但高速图有 4800~11000 个节点，图直径远大于 2。
s 与 t 的虚拟节点之间**信息传不到**（欠传播，under-reaching），InterGNN 的"跨区消息传递"事实上失效，
只能靠 `highway_dist_feat` 的显式分解距离作为骨架，纠偏能力受限。

### 涉及文件
- `main.py`：暴露 `--num_inner_layers` / `--num_inter_layers` CLI 参数并透传。
- `infer_distance.py`：加同名 CLI 参数（推理必须与训练一致）。
- `gnn.py` / `model.py`：`num_inter_layers` 已有构造参数，不用改。

### 思路
默认 `--num_inter_layers 4`。业内深图 GNN 甜点区在 4~6 层（GCNII / GraphGPS / DeepGCNs），
> 6 层容易触发 over-smoothing、需要更多正则。

### 预估收益
中大 —— Exp-5 曲线在 best_ep=115 仍在下降，说明模型远未饱和；InterGNN 是欠传播主战场，
理论上可解锁一层 rel_err 的空间。

### 风险
- 训练时间约 +30%（每轮 ~590s → ~770s）。
- **必须配 A2（residual/norm）**，否则梯度问题会让深层 GNN 直接不收敛。

## A2. Pre-norm + Residual + GraphNorm 块（深层可训） ✅ v1.0.0 引入 → v1.0.1 修复

> **重要修订（v1.0.1）**：v1.0.0 原方案用的是 `nn.LayerNorm`，Exp-6 实测 rel_err 从 0.119 崩到 0.824。
> 根因是 LN 沿每个节点自己的特征列归一化，把 `x_norm/y_norm`（节点位置）逐节点抹到零均值单位方差，
> GNN 失去空间锚点。**v1.0.1 已改用 `torch_geometric.nn.GraphNorm`**——图内跨节点归一化 +
> 每维可学习均值缩放 α，保留节点间的相对位置差异。下文思路块中的 `LayerNorm` 已过时，
> 实际实现见 `gnn.py` 中的 `InnerGNN._residual_stack` / `InterGNN._residual_stack`
> （签名多一个 `batch` 参数）。

### 动机
标准堆叠 `conv → ReLU → dropout` 在层数 > 3 时会梯度消失 / over-smoothing。DeepGCNs (ICCV'19)、
GCNII (ICML'20) 都证明 residual + norm 是深化 GNN 的必备组件。

### 涉及文件
- `gnn.py`：`InnerGNN` 与 `InterGNN` 的 `__init__` / `forward` / `forward_batch` 都要改。

### 思路（v1.0.1 最终版）
把当前的层循环改为 pre-norm residual 块（GraphNorm 版本）：
```python
h = self.input_proj(x)               # Linear: input_dim → hidden_dim
for i, conv in enumerate(self.convs):
    h_res = h
    h_normed = self.norms[i](h, batch)   # GraphNorm(hidden_dim)，需 batch 索引
    h_pre = self.act(h_normed)
    h_pre = dropout(h_pre)
    h_conv = conv(h_pre, edge_index)     # 每层都 hidden_dim → hidden_dim
    h = h_conv + h_res                   # 残差
h = self.output_proj(h)              # Linear: hidden_dim → output_dim
```
- 所有中间卷积统一 `hidden_dim → hidden_dim`（等宽），首尾各加一个 `Linear` 投影处理维度。
- **GraphNorm** 放在 conv 之前（pre-norm，稳定性优于 post-norm）。**注意 GraphNorm 必须传 batch 索引**
  区分不同图，否则跨样本节点会被混在一起归一化。
- 每层都有残差（含首层——因为 `input_proj` 已把维度对齐到 hidden_dim）。

### 预估收益
配合 A1，允许训练 4~6 层深；单独看 A2 对 2 层的收益很小。

### 风险
- 显存约 +15%（保留每层残差旁路的中间激活）；
- **旧 checkpoint 不兼容**（`state_dict` 结构变了，多了 `input_proj / output_proj / norms`）；
- **旧缓存不受影响**（缓存的是数据侧的 highway 上下文，与模型结构解耦）。

## A3-lite. 节点特征加 z 坐标 ⏳ 下轮候选

### 动机
边权 = 3D 欧氏距离含 z，但当前 `FeatureBuilder.node_row` 只用 `[label, degree, x, y]`——**z 完全没进模型**。
在地形起伏明显时，模型无法感知垂直方向的边权贡献，只能从坐标里推 2D 距离。

### 涉及文件
- `build_highway.py:build_mesh_graph`：`coords` 从 `(x, y)` 改为 `(x, y, z)`。
- `preprocess.py:FeatureBuilder`：加 `z_range / z_min / z_center`；`normalized_coord` 返回 3 元组；
  `node_row` 基础特征改为 `[label, degree, x, y, z]`（5 维）。
- `preprocess.py:build_synthetic_partition_inputs`：`s_global_feat/t_global_feat` 从 2D → 3D。
- `main.py`：`global_feat_dim` 从 2 → 3。

### 预估收益
在起伏明显的地形（EP_low 有一定起伏）上小-中；平地几乎无影响。

### 风险
- 特征维度变了 → **旧缓存不兼容**，需 `--no_cache` 重算；
- 旧 checkpoint 不兼容（`global_encoder` 首层维度变了）。

## A3-full. 边权直接注入消息传递（GINEConv/edge_attr） 💤 后续

### 动机
即使加了 z，边权仍然是**间接**给模型的。真正业内做法是把 `edge_attr` 喂进 conv：
GINEConv、TransformerConv、NNConv 都支持 `edge_attr` 参数，让消息聚合权重与边权相关。

### 涉及文件
- `gnn.py`：把 `SAGEConv` 换成 `GINEConv` 或 `TransformerConv(edge_dim=1)`；每层前向传 `edge_attr`。
- `preprocess.py`：构造样本时同时提供 `edge_attr = log1p(edge_w)` 张量。
- `build_highway.py:build_highway_context`：高速图也要有边权（transit 边权来自 `dist_from_boundary`，
  已有；只是没张量化）。

### 预估收益
中大（可能是最"根源"的架构升级）。但改动量大，需要 A1+A2 落地后再上。

### 风险
- 改动 ~100 行，涉及多文件；
- GINEConv 需要 `nn.Module` 参数（内层 MLP），要设计得合理；
- 旧 checkpoint / 缓存都不兼容。

## B1. 对称化输出（无向距离的自然约束） 💤 后续（低成本、锦上添花）

### 动机
无向距离必须满足 `d̃(s, t) = d̃(t, s)`，但当前 `DistancePredictor.forward` 按固定顺序拼接
`[h_s_inner | h_t_inner | h_s_inter | h_t_inter]`，输出不对称。README FAQ Q5 已列此局限。

### 涉及文件
- `gnn.py:DistancePredictor.forward` / `.forward_batch`：改 Fusion 输入的构造方式。

### 思路（两种做法可选）
1. **对称拼接**：把 `[h_s, h_t]` 换成 `[h_s + h_t, |h_s − h_t|]`（Siamese distance heads 常用）；
2. **对称读出**：`f([h_s, h_t]) + f([h_t, h_s])` 求平均，`f` 是 Fusion MLP。

### 预估收益
小-中。让 `d̃` 严格对称，通常泛化略提升；对审稿更是"README Q5 直接回答已解决"的加分项。

### 风险
- 改动 ~30 行；
- 输出语义微调后**旧 checkpoint 需重训**（Fusion 输入维度可能变）。
