# NeurSC 训练文档 2 —— 超参数消融实验

本文记录一组**训练超参数 / oracle 结构**的对照实验（在 `训练文档.md` 的基础流程之上）。
核心原则：**单变量控制（OFAT）**——一次只改一项，否则结果好坏无法归因。

> 基础环境、数据生成、单次训练命令见 `训练文档.md`。本文只讲"改哪些参数、怎么排序、怎么记录"。
> 下文命令以 **EP_low**（`dataset/EP_low/EP_low.off`，|V|=164238）为例，与首个 Baseline 日志一致。

## 2. 推荐实验顺序（OFAT，一次只改一项）

| 顺序 | 内容 | 其他参数 |
|---|---|---|
| 1 | 原始 Baseline | 全部保持原设置 |
| 2 | 只加入 Scheduler | 其余不变 |
| 3 | 比较不同初始 LR | Scheduler 保持相同 |
| 4 | 比较 Dropout | LR、Scheduler 固定为上一步赢家 |
| 5 | 检查 8:1:1 节点/边重叠（数据泄漏） | 不改训练模型 |
| 6 | 加入测试集重构（ε） | 先固定模型参数 |
| 7 | 比较不同 Epsilon | 只改重构强度 |

## 3. 可执行命令（Baseline + Exp-1~3，现在就能跑）

公共前缀（EP_low）：分区参数 `--max_depth 2 --capacity 512` 全程固定，缓存复用。

```bash
# —— Baseline：原始设置 ——
python main.py --off_file dataset/EP_low/EP_low.off --file_folder . \
  --max_depth 2 --capacity 512 --distance_samples 50000 \
  --learning_rate 0.001 --lr_scheduler none --dropout_ratio 0.2 \
  --num_epoch 100 --batch_size 16 --device cuda \
  2>&1 | tee logs/EP_low_baseline.log

# —— Exp-1：只加 Plateau scheduler（其余同 Baseline）——
python main.py --off_file dataset/EP_low/EP_low.off --file_folder . \
  --max_depth 2 --capacity 512 --distance_samples 50000 \
  --learning_rate 0.001 --lr_scheduler plateau --lr_patience 5 --lr_factor 0.5 \
  --dropout_ratio 0.2 --num_epoch 100 --batch_size 16 --device cuda \
  2>&1 | tee logs/EP_low_exp1_plateau.log

# —— Exp-2：在 Exp-1 基础上调低初始 LR 到 5e-4 ——
python main.py --off_file dataset/EP_low/EP_low.off --file_folder . \
  --max_depth 2 --capacity 512 --distance_samples 50000 \
  --learning_rate 0.0005 --lr_scheduler plateau --lr_patience 5 --lr_factor 0.5 \
  --dropout_ratio 0.2 --num_epoch 100 --batch_size 16 --device cuda \
  2>&1 | tee logs/EP_low_exp2_lr5e4.log

# —— Exp-3：在赢家 LR 基础上调低 dropout 到 0.1 ——
python main.py --off_file dataset/EP_low/EP_low.off --file_folder . \
  --max_depth 2 --capacity 512 --distance_samples 50000 \
  --learning_rate 0.0005 --lr_scheduler plateau --lr_patience 5 --lr_factor 0.5 \
  --dropout_ratio 0.1 --num_epoch 100 --batch_size 16 --device cuda \
  2>&1 | tee logs/EP_low_exp3_dropout01.log
```

> scheduler 生效时，日志每轮会打印 `lr=...`，降 LR 时额外打印 `lr reduced: a -> b`。
> Exp-4/5 需等 ε 稀疏化实现后补命令（届时加 `--spanner_epsilon` 或 `--transit_k`）。

---

## 4. ⚠️ 关键注意事项（务必遵守）

1. **测试集必须可比**：Baseline~Exp-3 在 **Full** test 集，Exp-4/5 在 **Reconstructed** test 集——
   两者指标**不能直接并排比"谁更准"**。正确做法二选一：
   - (a) 所有实验都在**同一个 test 集**上评估；或
   - (b) 每个模型**同时在 Full 和 Reconstructed 上各评一遍**，分两栏各自内部比。
2. **一次只改一项**：改多项无法归因。
3. **固定随机种子**：当前 `seed=42` 固定；要报方差需多种子重复（见 Roadmap）。
4. **test 点对对齐**：用 `main.py` 导出的 `<run>_test_pairs.csv` + `baseline.py` 保证同口径。
5. **日志归档**：每次训练 `2>&1 | tee logs/<实验名>.log`，文件名体现参数/版本。
