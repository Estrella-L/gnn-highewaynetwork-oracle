# NeurSC 训练文档 3 —— Exp-6：v1.0.0 网络架构 vs Exp-5 单变量消融

本次实验目的：**验证 v1.0.0（A1+A2）网络架构升级——加深 InterGNN 到 4 层 + Pre-norm Residual/LayerNorm
——相对 Exp-5 是否带来 rel_err 下降**。

> 前置阅读：`训练文档2.md`（超参数消融方法论）、`docs/CHANGELOG.md#v1.0.0`（架构改动细节）、
> `docs/待做.md`（本轮实施的 A1+A2 全文）。

---

## 1. 单变量对照说明

**唯一变量**：网络架构（v0.15 的 2 层裸堆叠 → v1.0.0 的 4 层 Pre-norm Residual）。
**其余所有参数与 Exp-5 完全一致**，否则不是干净对照。

| 项 | Exp-5（基线） | Exp-6（本次） | 是否变动 |
|---|---|---|---|
| `--off_file` | dataset/EP_low/EP_low.off | 同 | — |
| `--max_depth` | 3 | 3 | — |
| `--capacity` | 512 | 512 | — |
| `--distance_samples` | 50000 | 50000 | — |
| `--learning_rate` | 0.001 | 0.001 | — |
| `--lr_scheduler` | plateau | plateau | — |
| `--lr_patience / factor / min_lr` | 5 / 0.5 / 1e-6 | 5 / 0.5 / 1e-6 | — |
| `--dropout_ratio` | 0.1 | 0.1 | — |
| `--num_epoch` | 120 | 120 | — |
| `--batch_size` | 16 | 16 | — |
| `--num_inner_layers` | (硬编码 2) | **2**（显式传参） | 值不变 |
| `--num_inter_layers` | (硬编码 2) | **4** ⚠️ | **唯一变量** |
| InterGNN 结构 | 2 层 SAGEConv 裸堆叠 | 4 层 Pre-norm Residual + LN | 由代码升级引发 |
| InnerGNN 结构 | 2 层 SAGEConv 裸堆叠 | 2 层 Pre-norm Residual + LN | 由代码升级引发 |

> 严格说 v1.0.0 也把 InnerGNN 的层内块改成了 Residual/LN（尽管层数仍 2），所以"结构"这一项其实
> 是两处同时变——但都属于同一次架构升级。这是**架构消融**而非"只改 num_inter_layers"的纯参数消融。

**Exp-5 参考结果**：`test_mae=824.1, test_rmse=1214.8, test_relative_error=0.1187, best_ep=115`。

## 4. 正式实验 Exp-6 命令（EP_low，120 轮）

冒烟通过后，跑对照实验。

```bash
nohup python -u main.py \
  --off_file dataset/EP_low/EP_low.off --file_folder . \
  --max_depth 3 --capacity 512 \
  --distance_samples 50000 \
  --learning_rate 0.001 \
  --lr_scheduler plateau --lr_patience 5 --lr_factor 0.5 --min_lr 1e-6 \
  --dropout_ratio 0.1 \
  --num_epoch 120 \
  --batch_size 16 \
  --num_inner_layers 2 --num_inter_layers 4 \
  --early_stop_patience 15 \
  --highway_k 3 --inner_mode partition --loss_type log_l1 \
  --device cuda \
  > logs/EP_low_exp6_v1_inter4_120epoch.log 2>&1 &
```

### 预估时间
- 预处理：≈ 90 秒（命中缓存，加载高速上下文 + nearest_k 预计算）；
- 每轮：v1.0.0 加深 + residual 约比 Exp-5 慢 30%，从 ~590s → **~770s/轮**；
- 总训练：**≈ 26 小时**（120 轮跑满）或 **≈ 18 小时**（假设 early_stop 在 85 轮左右触发）。

如果 GPU 时间紧张、想先试 60 轮看趋势，把 `--num_epoch 60 --early_stop_patience 10` 即可。
但为了和 Exp-5（120 轮）严格对齐，**推荐还是跑完整 120 轮**。

## 5. 结果解读框架（跑完后照下表判断）

| 情况 | 判据（Exp-6 vs Exp-5） | 解释 |
|---|---|---|
| ✅ **显著提升** | rel_err ≤ 0.10 | A1+A2 命中——欠传播是真实瓶颈，4 层残差解锁长程感受野 |
| 🟡 **小幅提升** | 0.10 < rel_err < 0.119 | 架构升级有小收益，但可能不是主瓶颈；建议 A3-lite（加 z）继续压 |
| ⚪ **持平** | 0.115 ≤ rel_err ≤ 0.123 | 长程传播不是瓶颈（可能 `highway_dist_feat` 显式距离已覆盖），继续加层意义不大 |
| ❌ **变差** | rel_err > 0.13 | 出问题：可能 over-smoothing、或残差实现有 bug、或 lr 不适配深层 |

同时看**训练曲线**：
- 若 v1.0.0 的收敛**更快**（前 20 轮 val_mae 就到 1400 附近，Exp-5 要到 ep 34 才到），
  说明加深让模型更容易学到长程距离——即使最终 rel_err 差不多，也是"更好的表示"；
- 若曲线**同步**（Exp-5/Exp-6 前 60 轮几乎重合），说明前期瓶颈根本不在 GNN 层数，A1+A2 白做；
- 若 train_mae 下得比 val_mae 快很多、gap 快速扩大 → 过拟合，需要更强正则（dropout↑）或早停。

## 8. 一个诚实提醒

v1.0.0 是**架构升级**，不是"只调一个超参"。所以：
- **数值等价性不成立**：不像 A（预计算 nearest-k）或 scipy Dijkstra 那样只是等效计算路径。
  模型的每个权重、每次 forward 都会不一样。
- **收益不确定**：理论上 InterGNN 加深能解决欠传播，但**实际收益要看数据**。
  也许 `highway_dist_feat` 显式距离已经"抢走了"GNN 的活，加深收益就有限。
- **必须实验说话**：Exp-6 结果决定后续走向，不能拍脑袋。
