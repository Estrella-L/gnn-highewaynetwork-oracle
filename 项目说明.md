# NeurSC 当前项目说明（以可运行主线为准）

本文档基于当前代码实现，说明项目正在训练什么、损失函数是什么、模型结构是什么、训练与推理流程是什么。

---

## 1. 当前在训练什么

当前主线任务是**节点对最短路距离回归**（distance regression）。

- 训练入口：`main.py`
- 监督目标：从图中采样节点对 `(s, t)`，用 Dijkstra 计算真实最短路距离 `d(s,t)` 作为标签
- 模型输出：预测距离 `y_hat(s,t)`

换句话说，模型不是做分类，也不是做子图计数主训练；当前可运行训练主线是“给定两个点，回归它们在图上的最短路长度”。

---

## 2. 数据与样本是怎么构造的

### 2.1 图数据读取

- 输入格式：**`.off` 2-manifold 三角网格**（与 EAR-Oracle 一致）
- 读取与建图：`build_highway.load_off()` + `build_highway.build_mesh_graph()`
  - 节点 = 网格顶点，边 = 网格边，**边权 = 顶点间 3D 欧氏距离**
  - 同时产出全节点 (x,y) 坐标 `coords`
- 解析结果 `graph_info` 主要包含：
  - 节点 id、节点 label（网格无标签，置 0）、节点度
  - 边列表 `edge_u/edge_v` 与边权 `edge_w`
  - 邻接表 `neighbors`

### 2.2 监督样本构造

函数：`preprocess.build_distance_samples()`

- 无放回抽样**唯一** `(s,t)` 对 → 按源点分组 → 每个不同源点只跑一次 Dijkstra（对大网格高效，避免全 APSP）
- `--distance_samples` 为采样上限（`<=0` 表示尝试全部对；大图自动设保护上限）
- 网格场景默认 `undirected=True`（距离对称，只取 `s<t`）
- 真值距离为加权图最短路 `sample["distance"]`
- 再由 `split_distance_dataset()` 切分 train/val/test（输入已去重，无跨集泄漏）

### 2.3 每个样本的模型输入（三路输入）

函数：`preprocess.build_synthetic_partition_inputs()`

针对单个 `(s,t)` 样本会构造三部分输入：

1. **s 侧局部子图**：取 `s` 所在**四叉树叶子盒**的诱导子图，得到 `x_s, edge_index_s, s_idx`（对齐论文 G_i 分区；`--inner_mode ego` 可退回 2-hop ego 子图）
2. **t 侧局部子图**：取 `t` 所在叶子盒的诱导子图，得到 `x_t, edge_index_t, t_idx`
3. **高速图 highway**：
   - 由 `build_highway.build_pipeline_inputs()` 从 `.off` 派生（**真正的四叉树**分区 → 边界点高速）
   - `preprocess.build_highway_context()` 把派生结果张量化为固定上下文
   - 为 `s/t` 各选最近的 `k` 个 highway 节点（按**全图最短路**，非 id 近似）
   - 构造 `s_connect_idx, t_connect_idx`

节点特征为 `[label_norm, degree_norm, x_norm, y_norm]` 重复填充（坐标来自 `.off`，是真实位置信息）。

## 3. 模型结构是什么

核心模型：`gnn.DistancePredictor`（由 `model.DistanceRegressionNet` 包装调用）

整体是**三段式结构**：

1. **InnerGNN（局部段）**
   - 对 `s` 子图、`t` 子图分别用 GraphSAGE 堆叠编码
   - 取查询点位置的节点嵌入，得到 `h_s_inner`、`h_t_inner`

2. **InterGNN（跨分区段）**
   - 在 highway 图上再加两个虚拟节点（代表 `s` 与 `t`）
   - 虚拟节点由 `s_global_feat / t_global_feat` 经 MLP 编码得到
   - 虚拟节点分别连到各自的 highway 连接点集合
   - 经过 GraphSAGE 后取两个虚拟节点嵌入并做 readout，得到 `h_st_inter`

3. **Fusion MLP（融合回归段）**
   - 拼接 `[h_s_inner, h_t_inner, h_st_inter]`
   - 经多层 MLP 输出标量距离
   - 最后一层接 `Softplus`，保证预测距离非负

## 4. 损失函数和评估指标

### 4.1 训练损失

训练时在 `main.run_distance_epoch()` 中使用：

- `criterion = nn.L1Loss()`
- 即平均绝对误差（MAE loss）

形式：

- `L = |y_hat - y_true|`（批内取平均；当前实现按样本循环做单样本更新）

### 4.2 评估指标

`model.compute_distance_metrics()` 计算：

- `mae`
- `rmse`
- `relative_error = mean(|y_hat - y_true| / (y_true + eps))`

模型选择依据：验证集 `val_mae` 最小（早停 + 保存最佳参数）。

## 5. 训练流程（端到端）

入口：`python main.py --off_file <terrain>.off ...`

1. 解析参数（`.off` 路径、四叉树 `--max_depth/--capacity`、网络宽度、训练轮数等）
2. `build_pipeline_inputs`：读 `.off` → 建全局图 → **四叉树分区** → 派生高速节点/边/距离 → 高速上下文
3. 采样距离监督样本 `(s,t,d)`（加权最短路，无放回唯一对）
4. 切分 train/val/test
5. 初始化 `DistanceRegressionNet` + `Adam`
6. 训练循环：
   - 对每个样本构造三路输入
   - 前向预测距离
   - 按 `--loss_type`（默认 log_l1）反向传播更新
   - 每轮跑 train/val 指标
7. 按 `val_mae` 做 early stopping
8. 用最佳权重在 test 集评估
9. 保存模型与结果文件到 `saved_models/`、`saved_results/`、`saved_params/`

---

## 6. 推理流程（单对顶点）

入口：`infer_distance.py`

给定：

- 训练好的 `--model_path`
- 同一份 `.off`（`--off_file`）与**相同的四叉树参数**（`--max_depth/--capacity/--uniform`）
- 查询顶点 `--s --t`

流程与训练时单样本前向一致，输出 `predicted_distance(s->t)`。

---

## 7. 代码中“当前主线”与“历史模块”的关系

项目里还有 `Filtering`、`CoarsenNet`、`BasicCountNet`、`AttentiveCountNet` 等模块，偏向历史的子图匹配/计数路线。  
但**当前 `main.py` 实际训练路径只走 distance 分支**，核心依赖是：

- `main.py`
- `preprocess.py`（distance 样本与输入构造）
- `gnn.py`（DistancePredictor）
- `model.py`（DistanceRegressionNet + metrics）
- `infer_distance.py`（推理）

---

## 8. 当前实现特点与注意事项

- 输入为 `.off` 三角网格；全局图由网格构造，边权为顶点间 3D 欧氏距离（图最短路 ≈ 沿网格的折线测地距离）。
- 分区用**真正的四叉树**（自适应 / 均匀两种模式）；高速节点 = 盒子边界点；s/t 的最近高速入口按**全图最短路**选取（不再是 node id 近似）。
- 节点特征为 `[label_norm, degree_norm, x_norm, y_norm]`，坐标来自 `.off`，是真实位置信息。
- Inner-GNN 吃 s/t 所在**四叉树叶子盒的诱导子图**（对齐论文 G1~G4；`--inner_mode ego` 可退回 2-hop ego 子图做消融）。
- 距离监督为加权图最短路；样本是无放回唯一对、按源点分组算 Dijkstra，train/val/test 无跨集泄漏。
- 训练损失默认 `log_l1`（跨距离尺度归一化），输出层 `Softplus` 保证距离非负。
- 仍为逐样本更新（非 mini-batch）；Steiner 点 / WSPD spanner / Snell 加权测地距离尚未建模。
